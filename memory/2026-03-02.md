### PR #453 — Fix Unknown Payment Methods (zidpay + tabby)
- Branch: `fix/zidpay-payment-method`
- **Root cause:** Zid uses `"zidpay"` as a single generic code for ALL ZidPay-processed payments (Visa, Apple Pay, Mada, etc.). `PaymentMethod::tryFrom('zidpay')` returned null → UNKNOWN.
- Also confirmed: `"tabby"` is the code Zid uses for Tabby installments
- **Fix:** added `ZIDPAY = 'zidpay'` and `TABBY = 'tabby'` to `PaymentMethod` enum + AR/EN labels
- **Where code reads from:** `src/Integrations/Zid/Data/Order.php` line 375:
  ```php
  data_get($response, 'order.payment.method.code')
  ```
- In stored `order_info` JSON, raw code is at: `data.customer.details.orders[*].payment.method.code`

### PR #454 — Backfill Unknown Payment Methods Command

## Key Technical Findings

- Zid payment gateway `zidpay` covers: Visa, Apple Pay, Mada — all same code, different `name` field
- `tabby` (Zid code) ≠ `tabby_installment` / `tabby_pay_later` (existing enum cases)
- `order_info` stores serialized DTO (not raw API response) — `paymentMethod` in DTO already = `unknown`
  - To get raw code must dig: `order_info.data.customer.details.orders[*].payment.method.code`
